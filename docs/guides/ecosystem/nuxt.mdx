Bun 开箱即支持 [Nuxt](https://nuxt.com)。使用官方的 `nuxi` CLI 初始化 Nuxt 应用。

```sh terminal icon="terminal"
bunx nuxi init my-nuxt-app
```

---

要启动开发服务器，请在项目根目录运行 `bun --bun run dev`。这将运行 `nuxt dev` 命令（在 `package.json` 的 `"dev"` 脚本中定义）。

<Note>
  `nuxt` CLI 默认使用 Node.js；传入 `--bun` 参数则强制开发服务器使用 Bun 运行时。
</Note>

```sh terminal icon="terminal"
cd my-nuxt-app
bun --bun run dev
```

---

开发服务器启动后，打开 [http://localhost:3000](http://localhost:3000) 查看应用。应用将渲染 Nuxt 内置的 `NuxtWelcome` 模板组件。

开始开发你的应用时，可将 `app.vue` 中的 `<NuxtWelcome />` 替换为你自己的 UI。

生产构建时，虽然默认预设已兼容 Bun，你也可以使用 [Bun 预设](https://nitro.build/deploy/runtimes/bun) 来生成更优优化的构建。

```ts nuxt.config.ts icon="/icons/typescript.svg"
export default defineNuxtConfig({
  nitro: {
    preset: "bun", // [!code ++]
  },
});
```

或者，你也可以通过环境变量设置预设：

```sh terminal icon="terminal"
NITRO_PRESET=bun bun run build
```

<Note>
  有些包提供针对 Bun 的特定导出，Nitro 使用默认预设时不会正确打包这些导出。在这种情况下，你需要使用 Bun 预设，确保这些包在生产构建中能正常工作。
</Note>

使用 bun 构建完成后，运行：

```sh terminal icon="terminal"
bun run ./.output/server/index.mjs
```
